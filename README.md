# SiteSorter v2 - 現場フォルダ自動整理・現場管理アプリ

## 更新履歴

### 保守改善（セキュリティ・信頼性・UX）

- **セキュリティ強化**
  - rules.json のフォルダ名に `../` 等が含まれても現場フォルダ外へ書き込まないようパス検証を追加
  - 仕分け対象にシンボリックリンクが含まれる場合はスキップ（任意パスへの書き込みを防止）
  - settings.json / rules.json に 10MB のサイズ上限を設け、巨大ファイルによるメモリ枯渇を防止
- **エラー処理の改善**
  - 設定ファイルの「不在（初回起動）」と「破損」を区別し、破損時のみ警告を通知
  - 写真にEXIF撮影日が無くファイル更新日時へフォールバックした際にログへ明記
- **UXの改善**
  - Inbox一括仕分けの前に件数確認ダイアログを表示（Inboxが空の場合は早期通知）
  - 旧版アーカイブのダイアログをキャンセルしても、次回起動時にチェック状態を復元

## v2の主な変更（統合実装計画書_SiteSorter_v2.md 参照）

- フォルダ構成v2: 10_図面_作業用 / 11_図面_提出済（聖域）/ 12_社外受領データ / 13_図面_PDF / 21_3Dスキャン（01〜03）/ 99_Archive_旧データ を追加
- 13_図面_PDF: PDFはここに振り分け。ファイル名に「図面」「図」を含むPDFはキーワード優先で、それ以外のPDFも拡張子既定でこのフォルダへ（「報告書」等の既存キーワードに一致するPDFは従来どおり40_報告書・書類などが優先）
- 仕分け優先順位: トグル → キーワード辞書 → 拡張子 → 90_その他（複数マッチ・zipは要確認としてInboxに残る）
- ドロップゾーン＝即時仕分け、メイン画面の専用枠＝Inboxへ仮置き、の2経路
- サイドバーUI（ダッシュボード / 履歴・Undo / 設定）、テーマ2種切替（アース/ナイト）
- Undoは操作単位で直近50件（提出の同時コピーは2箇所とも削除）
- 設定・履歴・ルールの保存先は %APPDATA%\SiteSorter（旧配置から自動移行）

## セットアップ（初回のみ）

### 簡単な方法

1. `site_sorter` フォルダごと好きな場所（例: `C:\Tools\site_sorter`）に置く
2. `setup.bat` をダブルクリック（仮想環境作成→ライブラリ導入→自動テストまで実行）

※ Pythonが未インストールの場合は https://www.python.org/downloads/ から導入。
　インストーラーで「Add python.exe to PATH」に必ずチェックを入れること。

### 手動の場合

```bat
cd site_sorter
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

## 起動

`起動.bat` をダブルクリック（コンソール非表示で起動）。
ショートカットを作ってデスクトップやスタートアップに置くと便利。

手動の場合: `venv\Scripts\pythonw.exe main.py`

メインウィンドウとドロップゾーン（最前面の小窓）が起動します。

## 使い方

1. メインウィンドウの「参照...」で現場の作業フォルダを指定（次回起動時も記憶）
   → 00_Inbox〜90_その他 のフォルダが自動作成される
2. デスクトップのファイルをドロップゾーンへD&D → 00_Inbox へ移動
3. 「仕分け実行」→ 拡張子ルールに従い各フォルダへ移動（写真は撮影日 YYYY-MM-DD サブフォルダ）
4. 誤操作時は「直前の仕分けを元に戻す」（直近5バッチまで保持）

- 同名ファイルは `_YYYYMMDD_HHMMSS` 付きで保存（上書きなし）
- ドロップゾーンは左ドラッグで移動（位置を記憶）、右クリックでメニュー

## 複数現場の切り替え

- 一度使った現場はドロップダウンに履歴として残る（最大10件）。選ぶだけで切替
- ドロップゾーンの右クリック「現場を切り替え」からも切替可（●が現在の現場）
- ドロップゾーン下部に現在の現場名を常時表示（誤投入防止）

## タスクトレイ常駐

- 起動するとトレイにアイコンが入る。メインウィンドウを×で閉じても常駐は継続
- トレイアイコンのクリックでメイン表示、右クリックで「ドロップゾーンを表示/隠す」「終了」
- アプリの終了はトレイまたはドロップゾーンの右クリックメニューから

## 仕分けルールの編集

- メイン画面の「ルール編集」からフォルダ名・対象拡張子をGUIで変更可能（rules.json に保存）
- 行の追加・削除でフォルダ自体の増減も可能。00_Inbox / 90_その他 は固定
- .jpg/.jpeg を含むフォルダが撮影日サブフォルダの対象になる
- rules.json を削除すれば既定ルールに戻る
- 既存のrules.jsonにも、起動時に13_図面_PDF（フォルダ・キーワード）が自動追加される（既存設定は維持。.pdfが他フォルダの拡張子ルールに残っている場合はそこから除外）

## exe化（配布用）

`build.bat` をダブルクリック → `dist\SiteSorter.exe` が生成されます。

- 設定類は `%APPDATA%\SiteSorter` に保存されるため **exe単体で配布可能**（Python不要のPCでも動作）
- アイコン・qtawesomeフォントは同梱済み（`--add-data assets` / `--collect-data qtawesome`）
